# 🏫 수학 단원평가 제작소 (Math Unit Test Maker)

> **수학 교육과정 성취기준과 인지적 오개념 분석에 기반하여 고품질 단원평가 문항과 교사용 교수학습 피드백지를 맞춤 설계하고, AI 기반 사전 체크테스트를 통해 학생의 학습 상태를 정밀 진단하는 차세대 수학 교육 보조 웹 애플리케이션입니다.**

---

## 📖 1. 개발 배경 및 목적
수학 교사, 예비 교사, 그리고 교육 전공자들은 학생들의 학업 성취도를 정확하게 진단하고 이에 맞춘 평가지를 제작하는 데 많은 수고를 들입니다. 그러나 실무에서는 다음과 같은 큰 어려움에 직면합니다:
- **평가 설계의 높은 복잡도**: 학업 성취도에 알맞은 문항 수, 난이도 분포, 문항 유형(객관식, 단답형, 서술형)을 국가 교육과정 성취기준에 명확히 일치시키며 정교하게 설계하기란 매우 어렵습니다.
- **오개념 예방 설계의 리소스 부족**: 학생들이 평가 과정에서 겪는 인지적 오개념(예: 분모가 다른 분수를 더할 때 분모끼리 더하는 오류 등)을 사전에 예측하여 보기를 설계하고 맞춤 피드백을 수립하는 데 엄청난 리서치 시간이 소요됩니다.
- **진단과 평가의 유기적 연동 부재**: 단편적인 모의고사나 퀴즈만으로는 학생의 구체적인 약점을 정밀 진단하여 그 결과를 즉시 후속 단원평가 출제 조건에 반영하기 어렵습니다.

**“수학 단원평가 제작소”**는 이러한 문제를 교육 공학적으로 완벽히 극복합니다. 학생의 수준을 3~5개의 **AI 사전 체크테스트**로 신속 진단하여 **취약 개념 분석서**를 발급하고, 이 결과를 단원평가 출제 조건에 **⚡원클릭으로 자동 연동**합니다. 이를 통해 교사는 학생 개인별 맞춤 피드백이 완비된 고품격 교사용 해설지와 학생용 시험지를 클릭 한 번으로 손쉽게 출제할 수 있습니다.

## ✨ 2. 주요 기능

### ⚙️ [1] 직접 조건 설정 기반 단원평가 생성
- **학년별 국가 교육과정 맞춤 템플릿**: 초등학교, 중학교, 고등학교 학교급을 선택하면 이에 부합하는 표준 단원명, 세부 개념, 실제 교육과정 성취기준 및 적절한 평가 목적 템플릿이 자동으로 가득 차 편리한 출제가 가능합니다.
- **세분화된 난이도 및 문항 유형 밸런싱**: 
  - 슬라이더를 통해 쉬움/보통/어려움의 난이도 비율을 100% 한도 내에서 정교하게 조율할 수 있습니다.
  - 객관식(선다형), 단답형, 서술형의 비율을 직관적인 비주얼 게이지 바를 보며 설계할 수 있습니다.
  - '균등분배' 헬퍼 버튼을 통해 국가 표준 문항 설계 비율로 원클릭 셋업이 가능합니다.

### 🧪 [2] 체크테스트 기반 학습 진단
- **100% 객관식 사전 체크테스트 출제**: 학생의 취약 부위를 빠르게 진단하기 위해 3~5문항의 사전 퀴즈를 전격 객관식 5지선다 형태로 맞춤 설계하여 출제합니다. 
- **인터랙티브 답안 선택 및 정밀 채점**: 학생이 화면에서 ① ~ ⑤ 객관식 버튼을 직접 클릭하여 답안을 마킹하면, 제출 버튼 클릭 시 AI가 정밀 채점 및 개념 대조를 단 2초 만에 수행합니다.
- **오개념 분석 보고서 발급**: 
  - 보완이 시급한 취약 세부 개념 리스트 태깅
  - 학생이 해당 오답을 고른 구체적인 인지 오류(오개념)에 대한 교육학적 원인 분석
  - 후속 학습 및 보충이 필요한 사유를 명확히 설득해 주는 처방 가이드 제공

### ⚡ [3] 진단 결과 기반 출제 조건 자동 반영
- **진단 조건 원클릭 연동 (`⚡ 진단 결과로 출제 조건 적용하기`)**: 
  - 분석 보고서 하단의 적용 버튼을 누르면 AI가 추천하는 최적의 단원평가 출제 기준(학년, 취약 단원 세부 개념명, 관련 성취기준, 문항 수, 난이도 조절 비율, 문항 유형 가중치 등)이 **좌측 조건 설정 패널에 완전히 자동 전송 및 반영**됩니다.
  - 교사는 진단서를 기반으로 보정된 출제 셋업을 그대로 이어받아 학생에게 100% 정조준된 보정용 단원평가를 즉시 생성해 줄 수 있습니다.

### 🎓 [4] 학생용/교사용 보기 1:1 동적 전환 및 인쇄
- **학생용 문제지 뷰**: 학년/반/번호/성명 인적사항 기재 란, 표준 문제 지문, 5지선다 보기 레이아웃 및 깔끔한 풀이/정답 기입란이 깔끔하게 표시됩니다.
- **교사용 해설지 뷰**: 모범 답안(`🔑 올바른 모범 정답`), 교수법 통찰이 담긴 해설(`💡 풀이 과정 및 해설`), 학생 지도 방침(`⚠️ 예상 오개념 분석 & 피드백 방향`)이 컬러 카드로 표현됩니다. 추가로 **문항별 신속 정답 대조표** 및 **교육 전문가 코멘트**가 삽입됩니다.
- **A4 프린트 최적화 (`Ctrl + P` 지원)**: 시험지 출력 시, 네비게이션 헤더 및 설정용 사이드바가 화면에서 완전히 감추어지고 교과서 규격의 깨끗한 A4 실물 시험지만 맞춤 출력됩니다.

### 📐 [5] 완벽한 세로 분수(Fraction) 표시 개선
- **KaTeX/react-katex 의존성 완전 배제**: 브라우저 사양 및 기기에 따라 로딩 속도가 저하되거나 찌꺼기 텍스트가 깨져서 보이는 KaTeX 방식 및 교과서 가독성을 저해하는 유니코드 분수 기호(½, ⅔)를 전면 폐기하였습니다.
- **자체 HTML/CSS 기반 Fraction 컴포넌트 설계**: 평문 분수 표기법(`3/8`)이나 대분수 표기법(`2 1/4`)을 감지하여, 분자는 위에 가로선은 중간에 분모는 아래에 배치되는 깔끔한 **Textbook-Style 세로 분수**로 렌더링합니다.
- **유연한 룩어라운드(Lookaround) 파서**: 날짜(`2026/05/28`)나 파일 경로, URL의 슬래시(/)는 완벽히 보호 격리하여 그대로 유지하고, 순수 수학적 분수 텍스트만 정교하게 파싱해 냅니다. 혹시나 기존에 생성된 LaTeX의 잔재(`\frac{3}{8}`, `2\frac{1}{4}`)가 유입되더라도 예외 없이 무결하게 감지 및 보정해 내는 강력한 하이브리드 파서를 탑재했습니다.

## 🚀 4. 로컬 실행 방법
본 프로젝트는 Vite의 로컬 프록시 Connect 미들웨어를 내장하고 있어, 별도의 로컬 API 서버를 이중으로 띄울 필요 없이 하나의 터미널에서 프론트와 API 핸들러가 자동으로 프록시 연동됩니다.

### 1) 패키지 종속성 설치
```bash
npm install
```

### 2) 로컬 개발 서버 구동
```bash
npm run dev
```
- 터미널에 노출되는 개발 주소(기본: `http://localhost:5173`)로 접속합니다.
- 사전 진단 체크테스트 진행, 진단 결과 연동, 그리고 맞춤 단원평가 생성 기능까지 실제 개발 환경에서 실시간 조작이 가능합니다.

### 3) 프로덕션용 빌드 검증
```bash
npm run build
```
- 빌드 명령 수행 시 TypeScript의 타입 정밀 검사(`tsc -b`)가 가동되고, Vite 번들러가 정적 자원 및 서버리스 API 핸들러들을 완벽하게 최적화하여 빌드해 냅니다.

## ⚙️ 5. 환경변수 설정 방법
OpenAI API를 안전하게 사용하기 위해 프로젝트 루트 디렉토리에 환경변수 설정이 필요합니다.

1. 프로젝트 최상위 루트 디렉토리에 `.env.local` 파일을 생성합니다.
2. 발급받은 OpenAI API Key를 아래 형식에 맞추어 등록합니다.
   ```env
   OPENAI_API_KEY=여러분의_OpenAI_API_KEY_입력
   ```

> ⚠️ **보안 및 설계 준수 사항**
> - **실제 OpenAI API Key 값은 절대로 README.md나 소스코드 및 퍼블릭 GitHub 저장소에 유출되지 않도록 엄격히 은닉해야 합니다.**
> - `.env.local` 파일은 이미 `.gitignore`에 자동 등록되어 있으므로 소스코드가 원격에 업로드되어도 키는 절대 안전합니다.
> - 클라이언트 사이드 변수(`VITE_` 접두사)를 쓰지 않고 오직 서버 사이드 Serverless Functions에서만 비밀 환경변수로 로드되므로, 브라우저 탈취 등의 보안 위협으로부터 100% 안전합니다.

## 🌐 6. Vercel 배포 방법
본 프로젝트는 Vercel 플랫폼 환경에 즉시 배포할 수 있는 최적의 구성을 갖고 있습니다.

1. **Vercel 연동**: Vercel에 가입하고 본 프로젝트의 GitHub 저장소를 가져와 연동합니다.
2. **환경변수 설정 (Environment Variables)**:
   - Vercel의 프로젝트 설정(Project Settings) -> **Environment Variables** 메뉴로 이동합니다.
   - Key에 `OPENAI_API_KEY`, Value에 발급받은 실제 OpenAI API Key를 등록하고 저장합니다.
3. **빌드 설정 검증**:
   - Vercel의 Auto-detection에 의해 Vite 빌드 환경(`Build Command: npm run build`, `Output Directory: dist`)이 자동으로 셋업됩니다.
   - 저장소를 메인 브랜치에 Push하면 서버리스 API 함수들(`/api/*`)과 정적 웹 에셋이 완벽한 시너지를 내며 글로벌 상용 배포가 완료됩니다.
